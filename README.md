لح  اشرح نظام المخزن المؤقت الدائري (Circular Buffer) بلغة C 
هذا المشروع يوضح كيفية بناء وإدارة مخزن مؤقت دائري (Ring Buffer)، وهو هيكل بيانات فعال جداً لإدارة تدفق البيانات (Data Streams) خاصة عندما تكون موارد الذاكرة محدودة.
 1. بنية البيانات (Structure Definition) 
بدأنا بتعريف نوع بيانات مخصص CircularBuffer باستخدام struct ليحتوي على:
buffer[SIZE]: المصفوفة المادية التي تخزن البيانات (حجمها 12 بايت).
 head: مؤشر (Index) يشير إلى موقع أول عنصر جاهز للقراءة.
tail: مؤشر يشير إلى الموقع التالي الذي سيتم الكتابة فيه.
count: متغير لمتابعة عدد العناصر الحالية، وهو ضروري لمعرفة حالة الامتلاء أو الفراغ بسهولة.
 2. العمليات الأساسية (Core Functions) 
لقد قمت بتقسيم المنطق إلى دوال صغيرة لضمان سهولة الصيانة (Modularity):
 init: تقوم بتصفير كافة المؤشرات والعدادات لبدء تشغيل آمن.
 isFull & isEmpty: دوال منطقية تعيد قيم true أو false للتحقق من حالة المخزن قبل إجراء أي عملية.
write (الكتابة):
    تقوم أولاً بفحص "تجاوز السعة" (Buffer Overflow).
   المعادلة الجوهرية هنا هي (tail + 1) % SIZE. استخدام معامل باقي القسمة (%) هو ما يجعل المؤشر يعود للبداية (الصغر) بمجرد وصوله لنهاية المصفوفة، مما يحقق المفهوم "الدائري".
 read (القراءة)
    تقوم بفحص "النقص" (Buffer Underflow) للتأكد من وجود بيانات.
    تستخدم نفس المنطق الدائري لتحريك مؤشر الرأس head.
 3. منطق البرنامج الرئيسي (Main Execution Flow Loop) 
في دالة main تم تطبيق سيناريو عملي:
1. استلام البيانات: نأخذ اسماً من المستخدم ونقوم بدمجه مع سلسلة نصية ثابتة "CE-ESY" باستخدام strcat.
2. التخزين الحذر: نقوم بالدوران حول السلسلة النصية وكتابتها حرفاً بحرف. إذا زاد طول النص عن 12 حرفاً، سيعترض البرنامج تلقائياً ويطبع "Buffer Overflow"، مما يمنع الكتابة فوق مواقع ذاكرة غير مخصصة.
3. الاستخراج والعرض: في النهاية، نقوم بسحب كافة الحروف المخزنة وعرضها حتى يفرغ المخزن تماماً.
